概述

Pytest 是 Python 生态中最流行的测试框架之一，以「约定优于配置」著称：无需继承、少写样板，用普通函数和断言即可编写测试，支持参数化、夹具（fixture）、插件扩展，适合单元测试与接口测试。

特点：自动发现 test_*.py / *_test.py；断言失败时自动展示 diff；丰富的 fixture 与 marker；插件生态（覆盖率、并行、报告等）。

Pytest 通过「约定 + assert + fixture + 参数化」即可覆盖大部分测试需求。掌握测试发现、断言、fixture 作用域与 conftest、参数化与常用命令行选项，再结合项目需要选用插件（如覆盖率、并行、报告），即可高效编写和维护测试。

核心概念

概念 说明
测试函数 以 test_ 开头的函数，一个函数即一条用例
测试类 以 Test 开头的类，其内以 test_ 开头的方法为用例
断言 直接用 assert expr，失败时 pytest 会做表达式重写并输出详细信息
Fixture 用 @pytest.fixture 装饰的可复用准备/清理逻辑，通过参数注入到测试中

发现规则：默认收集当前目录及子目录下 test_*.py、*_test.py 中名为 test_* 的函数或 Test* 类中的 test_* 方法。

Fixture（夹具）

Fixture 用于准备数据、连接、环境等，在测试前执行、测试后可按需清理。

定义与使用

作用域

作用域 说明
function 每个测试函数执行一次（默认）
class 每个测试类执行一次
module 每个模块执行一次
package 每个包执行一次
session 整个测试会话一次

conftest.py 与共享 Fixture

将 fixture 写在项目或某目录下的 conftest.py 中，该目录及子目录下的测试无需 import 即可通过参数使用，便于共享（如公共的 client、db、config）。

参数化

同一套逻辑多组输入/期望，用参数化避免重复写多个测试函数。

可多组 @pytest.mark.parametrize 组合使用，形成笛卡尔积；ids 可指定每组用例的显示名称。

常用命令与标记

命令/选项 作用
pytest -v 详细模式
pytest -q 安静模式
pytest -x 首个失败即停止
pytest --lf 仅重跑上次失败的用例
pytest -n auto 多进程并行（需 pytest-xdist）
pytest --tb=short 简短 traceback
pytest -m "not slow" 按 marker 筛选

内置/常用 Marker 示例：

在 pytest.ini 或 pyproject.toml 中注册自定义 marker，避免告警。

易错

  1. 命名规范：测试文件 test_*.py，测试函数/方法 test_*，fixture 语义化命名（如 db、client）。
  2. 单一职责：一条用例只验证一个行为或一个场景，断言清晰。
  3. 用 fixture 做准备与清理：避免在测试里写大量 setup/teardown，提高可读性和复用。
  4. 公共 fixture 放 conftest.py：按目录划分（如 API 测试、单元测试）可分层 conftest。
  5. 合理使用 parametrize：相同逻辑多组数据用参数化；逻辑差异大时拆成多条用例更清晰。
