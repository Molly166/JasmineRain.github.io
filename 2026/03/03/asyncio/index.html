概述

asyncio 是 Python 3.4+ 内置的异步 I/O 框架，基于协程与事件循环，在单线程内实现并发，适用于 I/O 密集型场景（网络请求、文件读写、数据库访问等）。

核心思想：在等待 I/O 时让出控制权，由事件循环调度其他协程执行，从而避免阻塞、提高吞吐。

asyncio 通过 async/await + 事件循环 在单线程内实现 I/O 并发，适合高并发、低延迟的网络与 I/O 场景。掌握「协程 → Task → gather/run」这条主线，并避免在协程内使用阻塞调用，即可写出清晰、高效的无阻塞代码。

核心概念

概念 说明
协程 (coroutine) 用 async def 定义的异步函数，调用后返回协程对象，需在事件循环中运行
事件循环 (event loop) 负责调度、执行协程，在 I/O 就绪时唤醒对应协程
await 在协程内挂起当前协程，等待可等待对象完成，期间事件循环可执行其他协程

可等待对象 (Awaitable)：协程、asyncio.Task、asyncio.Future 等。

基本用法

定义与运行协程

  • async def 定义协程；直接调用 main() 只会得到协程对象，不会执行。
  • asyncio.run(main()) 创建事件循环、运行协程、结束后关闭循环，是运行「最外层」协程的标准方式。

并发执行多个协程

  • asyncio.create_task(coro) 将协程包装为 Task，加入当前事件循环并尽快调度，实现并发。
  • asyncio.gather(*aws) 并发执行多个可等待对象，全部完成后返回结果列表；任一异常会传播（可用 return_exceptions=True 收集异常）。

常用 API

API 作用
asyncio.run(coro) 运行顶层协程，管理事件循环生命周期
asyncio.create_task(coro) 创建 Task，并发执行
asyncio.gather(*aws, return_exceptions=False) 并发执行并汇总结果
asyncio.sleep(delay) 异步休眠，不阻塞线程
asyncio.wait_for(aw, timeout) 带超时的 await
asyncio.to_thread(func, *args) 在线程池中运行同步函数，避免阻塞事件循环 (3.9+)

与多线程的对比

维度 asyncio 多线程 (threading)
并发模型 协作式，单线程多协程 抢占式，多线程
适用场景 I/O 密集型 I/O 或 CPU 密集型（GIL 下 CPU 仍受限）
切换成本 协程切换成本低 线程切换、锁竞争成本较高
编程复杂度 需统一 async/await，避免阻塞调用 需处理锁、竞态

注意：在协程内调用同步阻塞函数（如 time.sleep、同步 HTTP 请求）会阻塞整个事件循环。应改用 asyncio.sleep、异步 HTTP 库（如 aiohttp），或用 asyncio.to_thread() 将阻塞调用放到线程池。

实践建议

  1. 入口统一用 asyncio.run()，避免手动 get_event_loop/run_until_complete。
  2. 需要并发时显式创建 Task：先 create_task 再 await，才能并发；若直接 await coro1(); await coro2() 则为顺序执行。
  3. 资源清理：用 async with 管理异步上下文（如 aiohttp.ClientSession），确保异常时也能释放。
  4. 异常处理：在协程内用 try/except；gather 时可按需使用 return_exceptions=True 统一处理异常与结果。
